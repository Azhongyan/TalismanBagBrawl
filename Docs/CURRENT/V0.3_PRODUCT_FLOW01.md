# 《符箓背包》V0.3-ProductFlow01 当前版本总蓝图

## 1. 定位

```text
V0.3-ProductFlow01 = 首次完整产品体验流转总蓝图
```

它不是一个单独开发包，而是 V0.3 各子包的总目标。

V0.3 要把 V0.2 灰盒核心循环包装为：

```text
有入口
有首页
有试练
有整备
有 Boss 前后
有锻造升级
能进入 2-1
```

---

# 2. 完整目标路径

V0.3 目标首登路径：

```text
Loading
→ StartGame
→ 服务器 / 区服占位
→ 开始游戏
→ OpeningStoryPanel
→ 首场教学战斗
→ BattleNormalState
→ 点击整备进入 BattlePrepareState
→ 整备态引导拖动道具到棋盘
→ 继续战斗
→ 1-1 到 1-10
→ BossNodeGuideState
→ BossInfoState
→ BossBattleState
→ BossResultState
→ 获得关键道具
→ 回首页
→ HomeForgeUnlockGuideState
→ ForgeUpgradeGuideState
→ 完成首次升级
→ 回首页
→ HomeTrialContinueGuideState
→ 点击试练进入 2-1
```

---

# 3. 底部导航最终口径

底部导航顺序锁定为：

```text
首页 / 探索 / 试练 / 锻造 / 更多
```

规则：

```text
梦签不进底栏
背包不进底栏
战斗背包只属于战斗页整备流程
梦签是首页中部空间热点
首页视觉包装为照灯小铺
```

# 7. 当前不可混做的内容

以下内容禁止放在同一个 Codex 任务里：

```text
BootEntryFlow + MainHome + BattlePrepare
MainHome + BossGuide + ForgeGuide
BottomNav + Battle Root
PrepareTutorial + FormationState 重构
Stage/Boss 触发 + 首页热点
```

原因：

```text
这些模块风险层级不同。
混做会导致 Codex 再次大包失控。
```
