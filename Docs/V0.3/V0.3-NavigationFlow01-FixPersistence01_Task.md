## 2. 失败现象

用户真实失败现象：

1. 从 `Scene_TalismanBag_V03_MainHome` 进入 `Trial`
2. 进入既有 `Scene_TalismanBag_V02_FormationCounter`
3. 推进至约 `1-5`
4. Stop Play
5. 再次从 V03 首页进入 `Trial`
6. 主线进度仍显示在约 `1-5`
7. 但棋盘上的道具消失，运行时道具状态像新开局，用户观察“道具记录像停在 1-1”

## 3. 修复目标

本修复包只解决：

- `Trial -> V02` 跨 Stop/Play 后的恢复一致性缺陷
- 让以下三者在跨会话恢复后保持一致：
  - 关卡进度
  - 运行时道具记录
  - 棋盘摆放状态

## 4. 非目标

本修复包不做：

- 任何首页导航功能扩展
- 任何 V03 页壳补做
- 任何正式炼符、探索、PVP、后屋玩法
- 任何奖励表、掉落表、数值表调整
- 任何战斗表现优化
- 任何 `PageState` / `FormationState` / `RunFlow` 重构
- 任何“只恢复库存不恢复布局”的降级交付

若产品要把目标降级为“只恢复库存、不恢复摆放”，必须另走产品契约变更，不得在本包内偷换目标。

## 5. 当前已知架构归因

固定 `Tech Architect` 的正式归因是：

- 主分类：`状态同步缺陷`
- 次分类：`既有 V02 持久化缺陷暴露`
- 补充：原任务档案 / QA 缺少“推进后 Stop/Play 再进入”的跨会话恢复硬门

当前已知真实链路：

- `V03NavigationFlowController` 只做 `LoadSceneMode.Single` 进入 V02
- `V02RunFlowController.Start()` 会进入 `StartNewRun()`
- `StartNewRun()` 会调用 `MainTrialFlowService.GetStartupRoute()` 恢复主线进度
- 同时 `StartNewRun()` 还会调用 `V02RewardController.StartNewRewardRun()`
- 当前运行时背包会被重置为固定 `StarterItemIds`
- 当前没有明确证据表明系统会按同一份持久化状态对称重建运行时道具与棋盘布局

## 6. 红灯说明

本包是红灯修复包，原因如下：

1. `V02RunFlowController` 属于受保护核心
2. 为满足“关卡进度、运行时道具、棋盘摆放三者一致”，可能需要碰 `SaveData / MainTrialProgressData` 或等价持久化结构
3. 修复范围若处理不当，可能破坏 V0.2 战斗黄金路径

因此：

- 本文档只能进入 `WAITING_GUARD_REVIEW`
- 后续必须先拿到 Guard 的明确 `GUARD_PASS`
- 再由用户明确审批具体红线文件范围
- 之后开发窗口才可恢复权限
