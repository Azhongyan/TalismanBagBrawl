# GOLDEN PATH QA

## 战斗黄金路径

- 进入战斗页能看到棋盘。
- 战斗中会跳伤害数字。
- 敌人血量会减少。
- 道具会触发。
- 供能状态可见且有效。
- 整备入口正常。
- 2-9 后 Boss 不自动开战。
- 2-10 Boss 必须手动触发。
- 奖励正常。
- 主流程能继续。

## 首页黄金路径

- 从任务档案规定的真实启动路径能进入首页。
- 如果任务要求独立新场景，必须验证实际进入该新场景，不能用旧场景中的隐藏面板替代。
- 首页默认处于任务档案约定的可见状态。
- 要求 Play 首帧可见时，测试和人工验证均不得先主动激活首页。
- Unity 或游戏重启后，首页仍能按任务档案约定的真实路径显示。
- 照灯小铺热点显示正常。
- 试炼入口进入当前主线。
- 炼符入口进入炼符页。
- 探索 / 后屋 / 梦签等未完成入口显示 ComingSoon 或锁定态。
- 二级页底栏显示正确。
- 底栏无“背包”或 `BattleBackpack` 入口。

## 使用规则

- 开发前根据改动影响选择必须验证的路径。
- 黄灯任务至少验证受影响页面路径。
- 任何可能触碰 V0.2 的任务必须验证完整战斗黄金路径。
- UI / 场景 / 页面任务必须同时验证代码存在与真实用户可见结果。
- 自动测试可以使用正式用户入口，但不得通过 reflection、临时实例化、直接切状态或主动激活目标来替功能完成被测前置条件。
- 自动测试若违反测试独立性，结果作废并按 QA 不通过处理。
- QA reviewer 必须列出用户在 Project、Hierarchy、Scene / Game View 中需要看到的具体证据。
- 任一核心不变量失败时立即停止，回滚本任务改动，不得继续叠加修复。
