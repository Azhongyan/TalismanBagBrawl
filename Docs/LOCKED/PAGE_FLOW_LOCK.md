# PAGE FLOW LOCK

## 页面职责

- 首页 = 照灯小铺空间化导航
- 探索 = 后续三选一 / 随机节点 / 秘境 / 天机炉方向；V0.3 当前只做占位或 ComingSoon
- 试练 = 主线自动战斗 / 巡行推进
- 锻造 = 包装现有第一次符箓培养 / 升级成长；V0.3 不重做完整锻造系统
- 更多 = 设置、公告、邮件、其他入口承载区
- 梦签 = 首页中部空间热点，不进入底栏
- 后屋 = 后续生产养成，不承担试练同类挂机收益
- 道藏 / 符箓库 = 图鉴 / 收藏 / 卡册，不承担战斗摆放
- 战斗背包 / 阵盘 = 试练和战斗流程内整备功能，不是一级导航页签
- 战斗页底部操作区 = 战斗内操作承载区，可在 V0.3-BattlePrepareInteraction01 中与当前整备口径合并；不是首页底部导航。
- 战后驻阵 = 用户可见口径统一为“整备”；这只锁 UI 文案 / 入口口径，不授权重写 RunFlow / FormationState。

## V0.3 底栏最终口径

V0.3 底栏顺序锁定为：

```text
首页 / 探索 / 试练 / 锻造 / 更多
```

规则：

```text
梦签不进底栏
背包不进底栏
战斗背包只属于战斗页整备流程
战斗页道具栏 / 整备入口不进入首页底栏
首页底部导航不直接搬进战斗页
梦签是首页中部空间热点
首页视觉包装为照灯小铺
底栏命名用“首页”，不是“小店”
锻造承接 1-10 Boss 后首次升级引导
```

本文锁底栏功能职责、命名口径、禁止项和 V0.3 顺序；不锁普通 UI 视觉细节。

不在本文中锁定：

- 尺寸、颜色、位置、间距
- 临时文案微调
- 图标表现
- 动效与灰盒布局形式

## 禁止

- 底栏出现“背包”
- 底栏出现 `BattleBackpack`
- 首页底栏承载战斗页道具栏 / 整备 UI
- 梦签进入底栏
- 道藏 / 符箓库承担战斗摆放
- 后屋承担试练挂机收益
- PVP 命名为“论道”
- 把“锻造”扩写成完整新锻造系统重做

页面职责、底栏功能边界或禁止项需要调整时，至少按黄灯处理；若调整会触碰战斗状态、Root 或 RunFlow，则按红灯处理。

仅调整视觉布局、颜色、尺寸、间距、临时文案或图标表现，且不改变页面职责、命名口径、底栏顺序与功能边界时，不视为修改本锁定文档。
