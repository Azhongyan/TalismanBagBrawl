# MEMORY FILE APPROVAL RULE

## 受保护文件

- `AGENTS.md`
- `Docs/LOCKED/*`

这些文件属于《符箓背包》的项目记忆与锁定文档，不是普通维护文档。

## 唯一治理窗口

建立本体系的原始 Codex 对话是唯一“记忆治理窗口”。

- 只有该原始窗口有资格提出并执行这些文件的修改。
- 所有其他新窗口、新线程、分叉线程、自动化线程和后续版本开发窗口只有读取权。
- 其他窗口不得创建、修改、追加、整理、格式化、覆盖、移动、重命名或删除受保护文件。
- 即使用户在其他窗口明确要求或批准修改，该窗口仍必须拒绝，并提示回到原始记忆治理窗口。
- 其他窗口不得自行声明继承治理权，不得通过复制对话、引用本文或读取历史记录获得治理权。

## 原始窗口的审批要求

原始记忆治理窗口也不得擅自修改。每次修改前必须向用户回报：

- 拟修改文件
- 修改原因
- 修改摘要
- 风险判断
- 是否影响 V0.2 稳定基线
- 是否改变当前版本范围、核心不变量或窗口权限

只有等待用户明确审批后才能修改。

## 未获审批时

- 不得修改。
- 不得自动整理。
- 不得主动重写。
- 不得以“看起来更规范”为理由改写。
- 不得将普通开发任务解释为更新项目记忆的授权。

## 权限变更

治理窗口转移、权限解除或本规则本身的修改，只能在原始记忆治理窗口中提出，并在用户明确审批后写入本文件。在此之前，任何其他窗口均无修改权。
