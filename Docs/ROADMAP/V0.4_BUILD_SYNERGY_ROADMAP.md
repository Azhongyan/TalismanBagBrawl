# 《符箓背包》V0.3/V0.4 BuildSandbox 路线图

## 1. 定位

```text
V0.3/V0.4-BuildSandbox = Build / 词条 / 羁绊 / 多格占位 / 验证池后台沙盒开发线
```

这条线适合在用户手动调整 UI、页面位置、背景、插槽和第一版美术风格期间并行推进。

它不是当前 V0.3 UI 主线，不是正式 Build 玩法上线，也不是正式多格背包接入。

## 2. 为什么现在做

当前用户正在 Unity 中手调：

```text
UI 插槽
页面位置
FullScreenBackground
第一版美术风格
首页 / 战斗页视觉结构
```

这段时间 Codex 不应继续频繁改页面和流程，也不应空等。

因此技术侧转向：

```text
后台沙盒
数据结构
验证工具
自动报告
devOnly 测试池
```

## 3. 总原则

```text
可以大步搭沙盒。
不能大步接正式流程。
```

所有新增能力默认：

```text
featureFlag = false
isEnabled = false
devOnly = true
```

## 4. 队列命名

正式队列名：

```text
V0.3/V0.4-BuildSandbox Technical Queue
```

当前执行队列：

```text
Docs/V0.4/BUILD_PACKAGE_QUEUE.md
```

含义：

```text
V0.3 阶段可以开始后台沙盒开发。
正式启用和内容化放到 V0.4 / V0.5 再判断。
```

## 5. Package 顺序

```text
Package 0：GuardBaseline01
Package 1：SynergyDataFoundation01
Package 2：ItemShapeOccupancy01
Package 3：SynergyEvaluatorCore01
Package 4：AffixRaritySandbox01
Package 5：ModifierEventBridge01
Package 6：BuildSimulationBenchmark01
Package 7：EnemyBossValidationPool01
Package 8：DevChapterContentPool01
Package 9：LedgerTaskBuildHooks01
Package 10：ConfigValidatorReport01
Package 11：FinalIntegrationDryRun01
```

`ItemShapeOccupancy01` 前置于 `SynergyEvaluatorCore01`。

原因：

```text
SynergyEvaluator 需要判断邻接、供能、位置、阵眼周围。
如果先做 Evaluator，再补多格占用，会返工。
```

## 6. 与当前 V0.3 主线关系

BuildSandbox 并行线不得阻塞或污染：

```text
V0.3 UI Runtime Lock
MainHome / BottomNav 手调
BattlePrepare UI Lock
Boot / Guide / Upgrade Runtime Lock
ForgeFirstUpgradeGuide01
ProductFlowRegression01
```

BuildSandbox 包不得修改当前 UI Scene / RectTransform / 手调布局。

## 7. 启动条件

当前已允许启动：

```text
V0.3/V0.4-BuildSandbox-GuardBaseline01
```

启动理由：

```text
GuardBaseline01 只建立沙盒护栏和验证入口。
它不接正式玩法、不碰 UI、不改主流程。
```

后续每包必须按 `Docs/LOCKED/BUILD_SANDBOX_BOUNDARY_LOCK.md` 收口。
